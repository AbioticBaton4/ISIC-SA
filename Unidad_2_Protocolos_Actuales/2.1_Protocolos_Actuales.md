# 2.1 Protocolos Actuales para Sistemas Abiertos (Tema 2.1)

## 2.1.1 Objetivos de los Protocolos Modernos
La meta fundamental de la estandarización lógica actual es garantizar la **interoperabilidad semántica**, abstraer la infraestructura subyacente y minimizar el *overhead* computacional y de red.

## 2.1.2 Paradigmas de Comunicación (Diferencias)
* **Request-Response (HTTP/3):** Comunicación síncrona iniciada por el cliente.
* **Pub/Sub (MQTT):** Comunicación asíncrona y desacoplada mediante un Broker central.
* **RPC (gRPC):** Ejecución remota de métodos utilizando contratos de datos estrictos.

## 2.1.3 - 2.1.5 Análisis Comparativo: Ventajas, Desventajas y Alcances
| Protocolo | Ventajas (Pros) | Desventajas (Contras) | Alcance Principal (Industria) |
| :--- | :--- | :--- | :--- |
| **HTTP/3 (QUIC)** | Multiplexación sin bloqueo, transiciones de IP fluidas, encriptación nativa. | Mayor carga de procesamiento en la configuración inicial; ineficiente para IoT ligero. | **Web Pública:** APIs REST, streaming de video, interfaces de usuario web. |
| **MQTT** | Cabeceras ultraligeras (2 bytes), bajo consumo energético, garantiza entrega en redes inestables (QoS). | Centralización en el Broker, inadecuado para transmisión de grandes volúmenes de datos o medios. | **IoT y Edge Computing:** Telemetría, ciudades inteligentes, automatización industrial. |
| **gRPC** | Velocidad extrema mediante serialización binaria (Protobuf), soporte para streaming bidireccional nativo. | Difícil depuración (cargas no legibles por humanos), requiere acoplamiento estricto del esquema de datos. | **Microservicios Internos:** Comunicación backend-to-backend masiva en centros de datos. |

## Conclusión para el Diseño de Red
La elección del protocolo dicta la topología y el rendimiento del sistema abierto. Un ingeniero selecciona MQTT para preservar el ancho de banda, gRPC para maximizar los ciclos de CPU en microservicios, y HTTP/3 para garantizar la accesibilidad global de los clientes.
